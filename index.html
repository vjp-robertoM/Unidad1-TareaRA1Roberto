Unidad 1 – Tarea RA1

Prueba y validación de una aplicación Python (Lavadero)

Objetivos de la actividad

El propósito de esta tarea es analizar, probar y validar una aplicación desarrollada en Python, aplicando buenas prácticas de documentación, depuración y pruebas de software.

En concreto, se trabajan los siguientes aspectos:

  • Análisis de la estructura interna del código (clases, métodos y flujo de control).
  • Comprensión del modelo de ejecución de la aplicación y sus transiciones de estado.
  • Uso de un IDE (Visual Studio Code) para la ejecución y depuración paso a paso.
  • Diseño e implementación de pruebas unitarias para validar los requisitos funcionales.
  • Detección y corrección de errores lógicos a partir de los resultados de los tests.
  • Ejecución de la aplicación en un entorno controlado (Sandbox).
  • Reflexión sobre la infraestructura de seguridad de distintos lenguajes de programación.

Resultados de aprendizaje y criterios de evaluación

Esta actividad contribuye al Resultado de Aprendizaje RA1, evaluando los siguientes criterios:

  • a) Análisis del código fuente y su comportamiento.
  • b) Uso de herramientas de desarrollo para ejecución y depuración.
  • c) Diseño y ejecución de pruebas unitarias.
  • d) Corrección de errores detectados mediante pruebas.
  • e) Ejecución controlada y reflexión sobre seguridad.

Secciones de la documentación

  1. Elementos de Python y documentación del código
    Código fuente de la aplicación lavadero comentado, explicando el propósito de clases, métodos y sentencias principales.

  2. Ejecución y depuración en el IDE
    Descripción de la ejecución de la aplicación en Visual Studio Code, errores detectados inicialmente y cómo se solucionaron, con capturas de pantalla.

  3. Pruebas unitarias
    Explicación de los tests realizados con Unittest/Pytest, relación de cada test con los requisitos del enunciado, errores detectados y resultados de la ejecución en modo verbose.

  4. Ejecución en entorno controlado (Sandbox)
    Ejecución de la aplicación en un entorno aislado, descripción del proceso y capturas de pantalla que evidencian el aislamiento.

  5. Reflexión sobre seguridad en lenguajes de programación
    Reflexión personal sobre las medidas de seguridad de distintos lenguajes de programación y comparación entre ellos.

Instrucciones de entrega

Para la correcta entrega de la actividad se ha realizado lo siguiente:

  • Creación del repositorio Unidad1-TareaRA1-Roberto en GitHub.
  • Inclusión de la estructura completa de MkDocs, requirements.txt y GitHub Actions.
  • Publicación de la documentación mediante GitHub Pages.
  • Inclusión de las carpetas:
  • src con el código fuente corregido.
  • tests con las pruebas unitarias realizadas.
  • Añadido el profesor PPSvjp como colaborador en modo lectura.

Finalmente, se entrega:

  • El repositorio comprimido.
  • El enlace a la página de GitHub Pages donde se visualiza esta documentación.
